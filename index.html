Become a Sandwhich

Become a Sandwich is a prototype dungeon crawler game. You play as the protagonist, the Bread Boy, a nd fight your way through a dungeon with your trusty crossbow fork launcher collecting upgrades and ingredients that strengthen your stats along the way. Your goal is to become a sandwich that is feared even by the evil tyrant, King Bread.

This was a group project for a game design course I took. I worked a lot on the enemy A.I., character stat systems, power-ups, weapon upgrades, and also designing a lot of the mechanics of the monsters and the boss. All the code was written in C# using the Unity game engine. Unfortunately we only had about 3 months to work on it, so the game is more of a demo. Working on this help me learn a lot about working on a large group project. Specifically how the work is split up and how efficient work pipelines are formed.

A.I. Denoising Medical Imaging

For this group project, we analyzed the effectiveness of a CNN A.I. Denoising network. We ran multiple tests on the Shepp-Logan phantom with different levels of Gaussian noise. For this project, we used MATLAB and the built-in machine learning technologies that it provided.

This project was very enlightening on the variety of uses that neural networks have on many different fields. Working with and understanding the internal structure of a CNN neural network inspired me to look further into A.I. and ML topics. Full code and report on GitHub

Fuel Price Prediction Website

A fellow classmate and I created a website for a Oil pseudo-company that would predict to price of an order of fuel deliver to a certain area. The website consisted of its own user log-in authentication system, a customizable user profile page, and a fuel price calculator.

The website was created with JavaScript, HTML, and CSS from scratch using PostgreSQL, Node.js, Express.js, and React.

We used Mocha and Chai for unit testing. I learned how creating a website without any website building services can be rather challenging and time consuming. I was very intrigued with all the layers (frontend, backend, and database) and how they all communicated with one another. Full code on GitHub

Markov Chain MIDI Generator

For this Max/MSP project, I used a Markov Chain to A.I. generate melodies based on an MIDI file of your choice. After the program reads a MIDI file, it feeds all the MIDI data (velocity, pitch, duration) into the Markov Chain in order to train it. Once you build it, the program generates a series of MIDI notes that resemble the orignal MIDI file, while staying orignal and unique.
